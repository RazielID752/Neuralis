# Como usar este cerebro

## Regra principal

Nao tente organizar tudo antes de estudar. Capture primeiro, refine depois.

## Fluxo diario

1. Coloque ideias soltas em [[Inbox/Entrada]].
2. Se uma ideia for importante, transforme em uma nota de conceito usando [[Templates/Conceito]].
3. Conecte a nota a pelo menos uma area em [[Atlas/Mapa Frontend]].
4. Crie um exemplo pratico em [[Laboratorio/Exercicios/Exercicios]] ou [[Laboratorio/Snippets/Snippets]].
5. Mova duvidas para [[Atlas/Perguntas Abertas]].

## Quando usar o Karpathy LLM Wiki

Use o plugin depois que houver algumas notas boas em `Conhecimento/Frontend`.

Sugestao:

- Ingerir primeiro uma area pequena, como `Conhecimento/Frontend/JavaScript`.
- Depois rodar em `Conhecimento/Frontend/TypeScript`.
- Por fim, ingerir `Conhecimento/Frontend/React`.

O resultado gerado pelo plugin fica em `wiki/`, separado das suas notas humanas.

## Padrao de qualidade para uma nota

Uma boa nota aqui nao precisa parecer uma aula publica. Ela precisa funcionar para voce reler e entender rapido.

Use uma juncao de dois estilos:

- caderno pessoal: direto, simples, com "o que eu preciso lembrar";
- explicacao didatica: do zero, com exemplos e pegadinhas.

Uma boa nota deve responder:

- O que e?
- Por que existe?
- Quando usar?
- Qual exemplo minimo?
- O que confunde iniciantes?
- Com quais conceitos se conecta?

## Formato recomendado

Use este formato quando for aprofundar qualquer tema:

1. `O que eu preciso lembrar`: resumo curto, do seu jeito.
2. `Explicando do zero`: sem assumir que voce lembra do basico.
3. `Por que isso existe`: qual problema resolve.
4. `Como funciona passo a passo`: processo mental.
5. `Exemplo bem simples`: menor codigo possivel.
6. `Exemplo mais real`: algo parecido com projeto.
7. `Pegadinhas`: coisas que costumam confundir.
8. `Resumo mental`: frase para fixar.
9. `Exercicios`: pratica curta.

# HTML

## O que eu preciso lembrar

HTML e a estrutura da pagina.

Ele nao existe para deixar bonito. Ele existe para dizer o que cada parte do conteudo significa: titulo, paragrafo, link, botao, formulario, navegacao, conteudo principal.

## Como eu vou estudar HTML

HTML parece simples, mas ele e a base de acessibilidade, SEO e estrutura.

O caminho e:

1. entender documento HTML;
2. usar tags pelo significado;
3. criar links, imagens e listas;
4. criar formularios;
5. pensar em acessibilidade desde o começo.

## Perguntas que eu sempre devo fazer

- Essa tag representa o significado certo?
- Isso e link ou botao?
- Esse input tem label?
- A pagina faz sentido sem CSS?
- A ordem dos titulos esta correta?

## Pegadinhas grandes de HTML

- Usar `div` para tudo.
- Usar link como botao.
- Usar botao como link.
- Pular de `h1` para `h4` por causa de tamanho visual.
- Criar input sem `label`.
- Colocar imagem importante sem `alt`.

## Resumo mental

HTML bom e aquele que continua fazendo sentido mesmo antes do CSS e do JavaScript.

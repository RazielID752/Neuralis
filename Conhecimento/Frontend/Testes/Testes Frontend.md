# Testes Frontend

## O que eu preciso lembrar

Teste serve para dar confianca de que o comportamento continua funcionando.

Teste bom nao testa detalhe inutil. Testa o que o usuario ou o sistema espera.

## Perguntas que eu sempre devo fazer

- Esse teste prova comportamento ou implementacao?
- Se eu refatorar, esse teste deveria quebrar?
- Esse fluxo e importante para o usuario?
- Estou mockando coisa demais?
- O teste e facil de entender?

## Pegadinhas grandes

- Testar so se componente renderiza.
- Testar classe CSS sem motivo.
- Mockar tudo e nao testar integracao real.
- Teste fragil que quebra com refactor simples.
- Nao testar estado de erro.

## Resumo mental

Teste bom me deixa mudar codigo com menos medo.

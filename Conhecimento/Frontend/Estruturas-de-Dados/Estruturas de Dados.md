# Estruturas de Dados

## O que eu preciso lembrar

Estrutura de dados e um jeito de organizar informacao para conseguir usar melhor depois.

Nao e coisa abstrata demais. No frontend eu uso estrutura de dados o tempo todo:

- lista de usuarios;
- objeto de formulario;
- mapa de erros;
- conjunto de ids selecionados;
- fila de notificacoes;
- pilha de historico.

## Explicando do zero

Se dado e baguncado, codigo fica baguncado.

Estrutura de dados responde:

- como vou guardar isso?
- como vou buscar?
- como vou adicionar?
- como vou remover?
- como vou evitar duplicado?
- como vou percorrer?

- Isso e uma lista?

- Preciso evitar duplicados?

- Preciso pegar o ultimo item?

## Pegadinhas grandes

- Usar array para tudo.
- Fazer busca com `find` toda hora quando um mapa por id seria melhor.
- Mutar array sem perceber.
- Nao saber se precisa de lista, objeto, map ou set.

## Resumo mental

Estrutura de dados e escolher o formato certo para o problema.

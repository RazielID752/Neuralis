# Tailwind CSS

## O que eu preciso lembrar

Tailwind e CSS usando classes utilitarias.

Em vez de escrever uma classe CSS grande, eu monto o visual direto no HTML/JSX com classes pequenas.

## Como eu vou estudar Tailwind

Tailwind so faz sentido se eu entender CSS por baixo.

O caminho e:

1. espacamento;
2. cores;
3. tipografia;
4. flex e grid;
5. responsividade;
6. estados;
7. tema;
8. componentes reutilizaveis.

## Perguntas que eu sempre devo fazer

- Eu entendo o CSS por tras dessa classe?
- Essa repeticao deveria virar componente?
- Esse layout funciona no mobile?
- Existe estado de hover, focus e disabled?
- Essa cor faz parte do sistema ou foi escolhida aleatoriamente?

## Pegadinhas grandes de Tailwind

- Usar Tailwind sem entender CSS.
- Criar linhas gigantes de classe sem componentizar.
- Esquecer `focus-visible`.
- Usar cor aleatoria em cada componente.
- Confundir breakpoint mobile-first.

## Resumo mental

Tailwind e uma forma rapida de escrever CSS, mas nao substitui entender layout, responsividade e estados.

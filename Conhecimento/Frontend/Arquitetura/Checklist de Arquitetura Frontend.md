# Checklist de Arquitetura Frontend

## Antes de criar uma pasta

- Essa pasta tem responsabilidade clara?
- Eu vou saber quando colocar algo nela?
- Existe outra pasta parecida demais?

## Antes de criar um componente

- O nome explica o que ele faz?
- Ele tem responsabilidade clara?
- Ele recebe props demais?
- Ele mistura UI com regra de negocio?

## Antes de criar um hook

- Essa logica usa recurso do React?
- Essa logica se repete?
- Uma funcao comum resolveria?

## Antes de buscar dados

- Esse dado e server state?
- Onde fica loading?
- Onde fica erro?
- Tem estado vazio?
- A chamada de API esta centralizada?

## Antes de abstrair

- Ja existe repeticao real?
- A abstracao deixa mais simples?
- Quem ler vai entender mais rapido?

## Resumo mental

Arquitetura boa reduz perguntas repetidas.

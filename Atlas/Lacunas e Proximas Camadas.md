# Lacunas e Proximas Camadas

## O que eu preciso lembrar

A base ainda nao e so sobre aprender sintaxe. Para virar conhecimento forte de frontend, ela precisa cobrir como construir, organizar, debugar, testar, otimizar e manter projetos.

## Camadas que ainda precisam crescer

### 1. Fundamentos da Web

Ja existe base inicial, mas ainda precisa aprofundar:

- DOM em detalhes;
- eventos do navegador;
- formularios avancados;
- acessibilidade pratica;
- SEO tecnico;
- imagens e performance;
- armazenamento no navegador.

### 2. JavaScript profundo

Ainda falta:

- event loop;
- prototypes;
- classes;
- imutabilidade;
- manipulacao de erros;
- modules em profundidade;
- fetch completo;
- abort controller;
- localStorage/sessionStorage;
- datas;
- regex.

### 3. TypeScript real

Ainda falta:

- discriminated unions mais profundas;
- type guards reais;
- tipos para API;
- schemas e validacao;
- `satisfies`;
- mapped types;
- conditional types;
- tipos para hooks e componentes genericos.

### 4. React de projeto

Ainda falta:

- data fetching;
- server state;
- context;
- reducers;
- hooks customizados;
- error boundary;
- performance;
- patterns de componentes;
- arquitetura de features;
- formularios com validacao;
- roteamento.

### 5. Arquitetura Frontend

Precisa cobrir:

- organizacao de pastas;
- separacao de responsabilidades;
- componentes UI vs feature;
- client state vs server state;
- camada de API;
- tratamento de erros;
- loading/empty/error states;
- padroes de escalabilidade;
- tradeoffs.

### 6. Testes

Ainda falta:

- testes unitarios;
- testes de componentes;
- testes de integracao;
- testes e2e;
- mocks;
- o que testar e o que nao testar.

### 7. Performance

Ainda falta:

- bundle;
- renderizacao;
- memoizacao;
- lazy loading;
- imagens;
- Core Web Vitals;
- profiling.

### 8. Ferramentas

Ainda falta:

- npm/pnpm;
- package.json;
- Vite;
- ESLint;
- Prettier;
- Git;
- DevTools;
- deploy.

### 9. Segurança

Ainda falta:

- XSS;
- CSRF;
- tokens;
- dados sensiveis;
- sanitizacao;
- headers;
- dependencia insegura.

## Como vamos evoluir

Para cada camada, criar:

1. mapa principal;
2. notas de conceitos;
3. exemplos pequenos;
4. exemplos reais;
5. pegadinhas;
6. exercicios;
7. checklist de revisao.
